주요 레슨런

🧠 생성형 AI API 처리 속도 최적화

  • Situation: AI 문제 풀이 기능에서 주관식 문제 처리 시간이 지연됨.
  • Task: 문제별 OpenAI API 요청을 최적화하여 속도를 개선해야 함.
  • Action: Coroutine을 활용해 주관식 문제 처리를 비동기 방식으로 변경.
  • Result: 문제 처리 속도가 약 15% 향상됨.

📡 운영 서버 모니터링 시스템 구축 및 오류 대응

  • Situation: 운영 서버에서 예기치 않은 오류 발생 시 즉각 대응이 어려움.
  • Task: 실시간 모니터링을 도입하여 오류를 빠르게 감지 및 대응해야 함.
  • Action: New Relic 기반의 모니터링 시스템 구축 및 APM을 통해 실시간 공유 설정.
  • Result: 반복 오류를 조기에 감지하고 해결하여 사용자 불편 최소화.

🔐 Redis 기반 Refresh Token 기능 구현

  • Situation: JWT 기반 Access Token 만료 시, 사용자의 반복 로그인 문제.
  • Task: 보안성과 효율성을 고려한 Refresh Token 시스템 설계 필요.
  • Action: Redis 기반의 Refresh Token 관리 및 RTR(Rotation) 기법 적용.
  • Result: DB 없이 JWT 검증 가능하게 하여 성능과 보안성 향상.
